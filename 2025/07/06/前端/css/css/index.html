选择器

简单选择题

选择器类型 语法 示例 描述与场景
标签选择器 element p { color: black; } 选中页面上所有的

标签。常用于设定全局性的、基础的元素样式。
类选择器 .classname .error { color: red; } 选中所有 class 属性包含 “error” 的元素。这是最常用、最灵活的选择器，用于组件、模块的样式定义。
ID 选择器 #idname #main-title { font-size: 32px; } 选中 id 属性为 “main-title” 的唯一元素。ID 在一个页面中必须是唯一的。由于其高优先级，应谨慎使用，通常用于页面级的主体结构或 JavaScript 锚点。
通配选择器 * * { box-sizing: border-box; } 选中页面上所有的元素。最经典的应用场景就是重置所有元素的盒模型（如示例所示）。注意： 因为会遍历所有元素，性能开销相对较大，需避免滥用。

高级选择器

组合方式 语法 示例 描述与场景
后代选择器 A B article p { color: #555; } 选中 article 元素内部的所有 p 元素，无论层级有多深。
子代选择器 A > B ul > li { list-style: none; } 仅选中 ul 元素的直接子元素 li。比后代选择器更精确，性能也更好，可以避免意外的样式应用。
相邻兄弟选择器 A + B h2 + p { margin-top: 0; } 选中紧跟在 h2 元素之后的第一个 p 元素。常用于调整连续元素的间距。
通用兄弟选择器 A ~ B h2 ~ p { font-style: italic; } 选中在 h2 元素之后的所有同级 p 元素。

伪类选择器

伪类类型 语法 示例 描述与场景
用户行为伪类 :hover, :focus, :active a:hover { color: red; } a:hover: 鼠标悬停在链接上时。
:focus: 输入框或按钮获得焦点时。
:active: 链接或按钮被点击的瞬间。
结构性伪类 :first-child, :last-child, :nth-child(n) li:first-child { font-weight: bold; } li:first-child: 选中作为父元素第一个子元素的 li。
li:last-child: 最后一个。
li:nth-child(2n): 选中所有偶数位的 li (例如实现表格的斑马条纹)。
逻辑伪类 :not(X) p:not(.fancy) { color: gray; } 选中所有不包含 .fancy 类的 p 元素。非常实用，可以减少样式覆盖。

伪元素选择器

伪元素 语法 示例 描述与场景
::before element::before p::before { content: “» “; } 在元素内容之前插入一个虚拟元素。必须包含 content 属性。常用于添加装饰性图标、符号。
::after element::after a::after { content: “🔗”; } 在元素内容之后插入一个虚拟元素。常用于清除浮动（经典的 clearfix hack）或添加图标。
::first-line element::first-line p::first-line { font-weight: bold; } 为块级元素的第一行文本设置样式。
::first-letter element::first-letter p::first-letter { font-size: 2em; float: left; } 为块级元素的第一个字母设置样式，常用于创建首字下沉效果。
::selection ::selection ::selection { background: #ffc107; color: #fff; } 为用户选中的文本设置样式。

优先级

优先级的计算规则（可以想象成一个四位数）：

  1. 内联样式 (Inline Styles): (1, 0, 0, 0) - 写在 HTML style 属性里的样式，优先级最高。
  2. ID 选择器: (0, 1, 0, 0) - 每个 ID 加 (0, 1, 0, 0)。
  3. 类、伪类、属性选择器: (0, 0, 1, 0) - 每个类（.class）、伪类（:hover）或属性选择器（[type="text"]）加 (0, 0, 1, 0)。
  4. 标签、伪元素选择器: (0, 0, 0, 1) - 每个标签（div）或伪元素（::before）加 (0, 0, 0, 1)。

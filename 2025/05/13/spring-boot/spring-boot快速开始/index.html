简单的项目结构

controller(访问控制层)

处理 HTTP 请求，接收客户端参数，调用 Service 层处理业务逻辑，并返回响应结果给客户端。

pojo(Plain Old Java Object)

定义数据模型，包括实体类（Entity）和数据传输对象（DTO）。

repository（数据访问层）

与数据库交互，提供数据持久化操作（增删改查）。

service（业务逻辑层）

实现核心业务逻辑，协调 Repository 层完成数据操作，并处理事务、权限等。
